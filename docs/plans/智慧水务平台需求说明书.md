## 二、项目概述

### 2.1 项目背景

随着城市化进程加快，水务行业面临供水规模扩大、设备老化、运营分散、数据孤岛等挑战。现有各水务子公司普遍存在以下问题：

- 设备台账分散，缺乏统一的设备全生命周期管理；
- 监测数据未集中管理，告警响应滞后；
- 生产运营依赖纸质记录，效率低下；
- 集团层面无法实时掌握各子公司运营状况，决策缺乏数据支撑；
- 安全隐患排查缺乏系统化管理工具。

为解决上述问题，建设一套覆盖设备管理、监测管理、生产管理、安全管理、物资管理、经营管理、管网管理、AI 智能助手的智慧水务平台，支持集团级多租户数据汇聚与跨公司对比分析，实现水务运营的数字化、智能化转型。

### 2.2 项目建设目标

1. **统一数据平台**：构建集团统一的数据管理平台，实现各子公司数据的集中汇聚与跨租户对比分析；
2. **设备全生命周期管理**：覆盖设备从登记、日常维护、缺陷处理、评估到大修更换的完整闭环管理；
3. **实时监测预警**：建立监测数据实时采集、分析、告警的完整链路，提升异常响应速度；
4. **生产运营规范化**：通过工单、值班、两票、交接班等数字化手段，规范生产运营流程；
5. **智能辅助决策**：引入 AI 智能助手，支持自然语言查询、异常预警分析和自动报表生成。

### 2.3 项目建设范围

**本期建设范围（Web 管理系统）：**

| 模块 | 说明 |
|------|------|
| 驾驶舱 | 集团看板 + 租户数据对比 |
| 设备管理 | 设备台账、缺陷、评估、项目管理、维护规程、展示分析 |
| 监测管理 | 监测配置、数据管理、视频监控、报警管理、数据分析、一张图 |
| 生产管理 | 工单、场景、两票、值班、交接班、工艺可视化 |
| 安全管理 | 隐患排查、安全巡检、应急预案、事故记录 |
| 物资管理 | 仓库、采购、库存、出入库 |
| 经营管理 | 项目管理、日/周/月报、计划执行分析、年度计划 |
| 管网管理 | GIS、漏损分析、巡检、档案 |
| AI 智能助手 | 智能对话、预警看板、数据分析报告 |

**不在本期范围：**

- 移动端 App（后续规划）
- 与外部 SCADA / IoT 系统的实时数据对接（本期使用 Mock 数据）
- 财务系统集成

**多租户支持：**

| 租户 | 说明 |
|------|------|
| 集团总览 | 汇聚所有子公司数据，展示汇总指标和跨租户对比 |
| 广州水务 | 独立管理广州分公司运营数据 |
| 深圳水务 | 独立管理深圳分公司运营数据 |
| 东莞水务 | 独立管理东莞分公司运营数据 |

### 2.4 业务流程概述

**多租户数据流向：**

各子公司租户独立维护各自的设备、监测、生产等业务数据。集团视角下，平台聚合所有子公司数据，提供汇总看板和跨公司对比分析。租户切换后，所有业务模块的数据自动联动刷新，展示对应租户的数据。

**核心业务流程：**

1. 用户登录后，顶部选择当前租户（集团总览或子公司）；
2. 驾驶舱展示当前租户的核心 KPI 概览；
3. 各业务模块按当前租户过滤数据，支持查询、维护操作；
4. 集团视角下，数据分析类页面展示多公司对比维度；
5. AI 智能助手在任意租户下均可使用，提供智能问答和报告生成。

### 2.5 用户角色定义

| 角色 | 说明 | 主要权限 |
|------|------|----------|
| 集团管理员 | 集团总部管理人员 | 查看所有租户数据，用户管理，系统配置 |
| 公司管理员 | 子公司管理人员 | 管理本公司所有业务模块，查看本公司所有数据 |
| 生产运营人员 | 一线运营人员 | 工单处理、值班记录、设备台账维护 |
| 设备维护人员 | 设备专岗人员 | 设备缺陷登记、维护保养执行、维护规程查阅 |
| 安全管理人员 | 安全专岗人员 | 隐患排查、安全巡检、应急预案管理 |
| 只读用户 | 报表查阅人员 | 查看各类报表和分析结果，无写操作权限 |

## 五、通用 UI 规范

- **布局**：左侧导航（240px）+ 顶部 Header（64px）+ 右侧内容区
- **颜色**：主色 `#409EFF`，成功 `#67C23A`，警告 `#E6A23C`，危险 `#F56C6C`
- **卡片**：`ElCard shadow="never"`，统一圆角和 padding
- **表格**：`stripe border size="small"`，分页统一 `[10, 20, 50]`
- **表单**：`label-width="100px"`，必填项红色星号，校验提示 inline
- **弹窗**：操作类用 `ElDialog width="600px"`，详情类用 `ElDrawer width="50%"`
- **按钮**：列表操作区 `type="primary" size="small"`，危险操作 `type="danger"`
- **标签**：状态标签统一用 `ElTag`，颜色与状态语义对应

## 六、Mock 数据规范

- Mock 数据放置于 `src/mock/modules/` 目录下，按模块分文件
- 所有接口支持 `tenantId` 参数过滤，集团视角返回聚合数据
- 分页接口统一返回 `{ list, total, page, pageSize }` 结构
- 状态枚举、人员列表等基础数据在 `src/mock/base.ts` 统一维护
