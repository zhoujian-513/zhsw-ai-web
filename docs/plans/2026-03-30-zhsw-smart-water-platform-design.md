# 智慧水务平台设计文档

**日期**: 2026-03-30
**项目**: zhsw-ai-web-v1
**技术栈**: Vue 3.5 + TypeScript + Element Plus + Pinia + Mock 模式

---

## 一、需求概述

构建一套智慧水务平台前端系统，支持逻辑多租户管理（水务公司/分公司），平台可在顶部切换租户，并支持集团视角下的数据汇聚（汇总展示 + 跨租户对比分析）。

---

## 二、多租户架构设计

### 租户类型
- **集团总览**：汇聚所有子公司数据，展示汇总指标和跨租户对比
- **子公司租户**：独立管理各自水务运营数据（如广州水务、深圳水务、东莞水务）

### 租户切换机制
- 顶部 Header 增加租户选择器下拉组件
- 当前租户状态存储在 Pinia store（`useTenantStore`）
- 切换租户后，所有页面数据通过 `currentTenantId` 联动刷新
- Mock 数据按 `tenantId` 区分，集团视角返回聚合数据

### Pinia Store 设计
```ts
// src/store/modules/tenant.ts
interface TenantState {
  currentTenantId: string   // 'group' | 'gz' | 'sz' | 'dg'
  tenantList: Tenant[]
  isGroupView: boolean       // 是否集团视角
}
```

## 四、各模块页面设计

### 4.1 驾驶舱
- **集团看板**：ECharts 多图表展示全集团 KPI（供水量、设备总数、告警数、收入等），按租户分组柱状图
- **租户对比**：多租户横向对比表格 + 雷达图

### 4.2 设备管理（最高优先级）
| 页面 | 主要功能 |
|------|----------|
| 设备编码配置 | 设备编码规则维护，树形结构展示 |
| 设备台账 | 设备列表（表格+搜索+分页），设备详情，新增/编辑/删除 |
| 设备缺陷 | 缺陷记录列表，缺陷状态流转（发现→处理→关闭） |
| 设备评估 | 评估任务列表，评估结果录入，评估报告查看 |
| 设备项目管理/维护保养 | 保养计划列表，执行记录 |
| 设备项目管理/年度中小修 | 年度检修计划，进度跟踪 |
| 设备项目管理/大修重置及技改 | 大修项目管理，技改申请审批 |
| 维护规程管理 | 规程文档列表，版本管理 |
| 设备展示分析 | 设备分布统计图表，运行状态分析 |

### 4.3 监测管理
| 页面 | 主要功能 |
|------|----------|
| 监测对象基础配置 | 监测点位配置，监测参数设置 |
| 监测数据管理 | 历史数据查询，数据导出，趋势图 |
| 视频监控管理 | 视频列表，在线预览（iframe模拟） |
| 报警管理 | 告警列表，告警确认，告警统计 |
| 数据分析管理 | 多维度分析，自定义报表 |
| 监测一张图 | 基于地图（china-map组件）的监测点位展示 |

### 4.4 生产管理
| 页面 | 主要功能 |
|------|----------|
| 运维工单管理 | 工单创建/指派/处理/关闭全流程 |
| 运维场景管理 | 运维场景配置，触发条件设置 |
| 两票管理 | 工作票/操作票列表，审批流程 |
| 值班管理 | 值班排班表，值班记录 |
| 交接班管理 | 交班记录，接班确认 |
| 工艺可视化 | 工艺流程图（SVG静态图+状态标注） |

### 4.5 安全管理
| 页面 | 主要功能 |
|------|----------|
| 隐患排查 | 隐患登记，整改跟踪，闭环管理 |
| 安全巡检 | 巡检计划，巡检记录，问题上报 |
| 应急预案 | 预案文档管理，演练记录 |
| 事故记录 | 事故登记，原因分析，处理结果 |

### 4.6 物资管理
| 页面 | 主要功能 |
|------|----------|
| 仓库管理 | 仓库信息，库位管理 |
| 采购申请 | 申请单列表，审批流程 |
| 库存台账 | 物资库存列表，库存预警 |
| 出入库记录 | 出入库流水，统计分析 |

### 4.7 经营管理
| 页面 | 主要功能 |
|------|----------|
| 经营项目管理 | 项目列表，合同管理，进度跟踪 |
| 生产运行信息管理/日报 | 日报录入，历史日报查看 |
| 生产运行信息管理/周报 | 周报汇总，自动聚合日报数据 |
| 生产运行信息管理/月报 | 月报分析，图表展示 |
| 计划执行分析 | 计划 vs 实际对比分析图表 |

### 4.8 管网管理

|------|----------|
