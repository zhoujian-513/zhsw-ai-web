# 【产品模块】应急抢修管理

## 一、业务概述
业务场景：应急管理人员对发生的应急事件信息进行初报，并根据后续处理情况和进展进行续报和终报
适用角色：应急管理人员
业务流程：初报 → 续报 → 终报
权限规则：不同角色可见按钮、可编辑字段不同

## 二、页面清单&交互链路
1. 列表页 → 点击“详情”详情跳转详情页
2. 列表页 → 点击”事件初报“ → 新增应急事件
3. 列表页 → 点击”事件续报“ → 对已初报的应急事件进行续报
4. 列表页 →  点击”事件续报“ → 对已初报的应急事件进行终报

## 三、页面1：数据列表页
### 页面布局
顶部搜索栏 + 状态筛选 + 功能按钮 + 数据表格 + 分页
### 搜索筛选项
所属单位、事件类型、事件状态、发生月份

### 表格展示字段
所属单位、事件类型、发生时间、地点/部位、核心不良影响、初步处置措施、事件状态、提交人、提交时间

### 页面按钮
续报、终报、详情、批量导出、批量导入

### 交互规则
- 状态不同颜色高亮：正常/预警/已过期
- 分页默认20条/页
- 点击单据行跳转详情
- 无权限按钮自动灰显隐藏

## 七、页面5：事件详情页
-只读展示全部信息
- 点击“日志”，可查看事件的日志记录列表，包括操作人、操作时间、操作类型（初报、续报、终报）
- 续报信息，可能存在多条，按续报时间列表展示

## 八、事件记录状态机
进行中：提交事件后，即为进行中，进行中的事件可以进行续报和终报；续报时，如果抢修处置进展选择为“持续处置中”，该事件状态仍然为“进行中”；
已结束：续报时，如果抢修处置进展选择为“已结束”，该事件状态变为“已结束”，或者点击“终报”按钮

## 九、权限控制
1. 应急事件管理员：全部初报、续报、终报记录权限
2. 普通业务：仅查看详情
